## Overview

Analyzed multi-table datasets using SQL to answer key business questions for an antiques store, covering employee compensation, sales performance, order trends, and training data insights.

---

## Problem

Raw relational data across multiple tables required structured querying to surface actionable business insights — from payroll benchmarking to sales and operational trends — that would otherwise be difficult to extract manually.

---

## Solution

- Wrote SQL queries using multiple join types to connect data across tables
- Used CTEs to organize and simplify complex query logic
- Applied window functions to perform calculations like identifying employees earning above-average salaries
- Analyzed sales, order, and training datasets to answer targeted business questions

---

## Methods Used

| Method | Purpose |
| :--- | :--- |
| Multiple Join Types | Connecting related tables across the dataset |
| CTEs | Organizing complex, multi-step query logic |
| Window Functions | Comparative calculations (e.g. above-average salaries) |
| Aggregate Analysis | Sales, order, and training data insights |
